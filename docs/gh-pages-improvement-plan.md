# GitHub Pages 개선 기획

> **목표:** 바이브코더가 30초 안에 “뭘 하면 되는지” 이해하고 프롬프트를 복사한다.  
> **원칙:** 쉽고 심플 — 스크롤·섹션·선택지를 늘리지 않는다.

---

## 1. 현황 (잘 된 점)

| 항목 | 상태 |
|------|------|
| 한 줄 프롬프트 + 복사 버튼 | ✅ 핵심 CTA 명확 |
| 3단계 미니 가이드 (복사→붙여넣기→완료) | ✅ 진입 장벽 낮음 |
| 8단계 워크플로 | ✅ `setup.prompt.md`와 정합 |
| 삭제 동의 / 화이트리스트 FAQ | ✅ 안전 메시지 반영 |
| 18+ 도구 칩 | ✅ 범위 전달 |

**진단:** 정보는 맞지만 **길고 반복**됨. Hero·비교표·8단계·FAQ가 비슷한 말을 여러 번 함.

---

## 2. 사용자가 실제로 겪는 갭

`setup.prompt.md` 기준으로, 사이트에 **아직 약한** 부분:

1. **진행 중 대화** — AI가 “스택은 Next.js”, “3번 빼줘”처럼 말한다는 예시 없음 → 첫 사용자 불안
2. **Rules vs Skills** — 둘 다 설치한다는 건 쓰지만 **한 줄 정의** 없음
3. **도구 감지** — “WHO AM I?”(지금 쓰는 채팅 = Cursor 등) 설명 없음 → “어떻게 아나?” 질문
4. **설치 위치** — Codex는 `.agents/skills/`만 쓴다 등 **흔한 오해** FAQ 없음
5. **8단계 = 스크롤 부담** — 바이브코더에게는 3문장 요약이 먼저 필요

---

## 3. 개선 원칙 (고정)

1. **한 화면 = 한 메시지** — 섹션 추가보다 기존 섹션 압축 우선
2. **행동 1개** — Primary CTA는 “프롬프트 복사”만
3. **기술 상세는 접기** — 8단계·레지스트리·경로표는 “더 보기”
4. **README / prompt와 문구 동기화** — 숫자·레지스트리 이름·안전 규칙 일치

---

## 4. 정보 구조 (제안) — 섹션 수 유지, 내용만 재배치

```
[Hero]  한 줄 가치 + 3-bullet (설치·정리·복붙)  ← 유지, 문장만 10% 짧게
[Start] 프롬프트 카드 + 3-step 미니  ← 유지
        + NEW: 「AI가 이렇게 물어봐요」대화 2~3턴 (접을 수 있음)
[Why]   Before/After 2칸  ← 유지 (8단계와 중복 문장 삭제)
[How]   NEW: 「3가지만 기억」 (분석 → 목록 확인 → 설치)
        + 기존 8단계 → <details>「자세히: 8단계」
[Tools] 칩 그리드  ← 유지
        + NEW: 인기 3종만 「파일이 생기는 곳」1줄 (Cursor / Claude / Codex)
[FAQ]   6개 → 7개 (+ Codex 경로, + 번호로 빼기)
[CTA]   복사 유도  ← 유지
```

**삭제·통합 후보 (심플함):**

- Hero `hero-detail` 2문장 → 1문장으로 합치기
- 8단계를 항상 펼쳐 두지 않고 **접기 기본**

## 5. 섹션별 상세 기획

### 5.1 Hero

| 변경 | 내용 |
|------|------|
| 추가 | 서브라인 1개: *「터미널 없음 · 레포 열린 채로 AI 채팅만」* |
| 수정 | H1은 유지, `hero-detail`은 스택 예시 1개만 (Next.js 나열 축소) |

### 5.2 Start (프롬프트 카드) — **최우선**

| 변경 | 내용 |
|------|------|
| **대화 미리보기** (신규, 카드 하단) | 3턴 고정 카피 (setup.prompt 톤과 동일): |

```
AI: 프로젝트가 Next.js + TypeScript 같아요. 기존 규칙 볼게요.
AI: 설치 목록 12개예요. 빼고 싶은 번호 있으면 말해주세요. 없으면 진행할게요.
나: 3번 빼줘
AI: 완료! 요약은 아래예요.
```

| 유지 | 복사 버튼, raw 링크, 4-step 미니 |
| 선택 | 모바일에서 **하단 고정 「복사」** (스크롤 후에도) — P1 |

### 5.3 How — **단순화 핵심**

**기본 노출 (3카드만):**

| # | 제목 | 한 줄 |
|---|------|------|
| 1 | 프로젝트 읽기 | package.json 등으로 스택 파악 |
| 2 | 목록 확인 | 맞는 규칙·스킬 골라 번호로 보여줌 → 빼고 싶은 것만 말하기 |
| 3 | 알맞은 폴더에 설치 | Cursor면 `.cursor/`, Codex면 `.agents/skills/` 등 |

**접힌 영역:** 기존 8단계 카드 그리드 (`<details open=false>`)

### 5.4 Tools

| 변경 | 내용 |
|------|------|
| 유지 | 카테고리별 칩 |
| 추가 | 「자주 쓰는 3곳」만 경로 1줄 (표 전체 X) |

- **Cursor** → `.cursor/rules/`, `.cursor/skills/`
- **Claude Code** → `.claude/skills/`, `CLAUDE.md` (덮어쓰기 안 함)
- **Codex** → `.agents/skills/`, `AGENTS.md` (`.codex/skills/` 아님)

### 5.5 FAQ

| 추가 질문 | 답 요지 |
|-----------|---------|
| AI가 내 도구를 어떻게 알아요? | 지금 대화 중인 제품(Cursor 채팅 등) + 프로젝트 폴더 자동 감지 |
| 3번 빼고 싶어요 — 어떻게 말하나요? | 설치 목록 번호만 말하면 됨 |
| Codex 쓰는데 `.codex/skills/`에 안 깔려요 | 정상. 공식 경로는 `.agents/skills/` |

기존 6개 유지. **레지스트리 FAQ**에 GitHub 링크 4개 추가 (이름만 → 클릭 가능).

### 5.6 메타·접근성 (가벼운 P1)

- `og:image` 정적 이미지 1장 (공유 시 미리보기)
- FAQ 버튼 `aria-expanded` (이미 토글 있음 → 속성만)
- `lang` 토글은 **v2** — 전체 i18n은 범위 밖 (README EN 한 줄 링크만 footer)

## 7. 카피 초안 (동기화용)

**Rules vs Skills (1줄):**

> **규칙** = “이 프로젝트에서는 이렇게 코딩해” (스타일·패턴) · **스킬** = “이런 일 할 때 이렇게 해” (작업 절차)

**3가지만 기억:**

> ① 프로젝트 읽기 → ② 설치 목록에서 빼고 싶은 번호만 말하기 → ③ 맞는 폴더에 자동 설치
